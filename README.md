# 📁 SMFM — Простой Файлообменник

> **Для чего это нужно?** Быстро передавать файлы между телефоном, компьютером и автомобильной магнитолой без флешек и проводов.

## 🎯 Что это такое простыми словами

Представьте: вы скачали APK-файл на телефон и хотите установить его на магнитолу в машине. Обычно нужна флешка или провод. С SMFM вы просто:

1. Открываете сайт на телефоне
2. Загружаете файл
3. Открываете тот же сайт на магнитоле
4. Скачиваете файл

**Всё!** Работает как личный Dropbox, но без регистрации, рекламы и ограничений.

## ✨ Возможности

- 📱 **Работает везде** — телефон, планшет, компьютер, магнитола
- 🔒 **Защищено паролем** — только вы имеете доступ
- 📂 **Папки** — организуйте файлы как удобно
- 🚀 **Большие файлы** — загружайте видео, игры, APK любого размера
- 💾 **Контроль места** — установите лимит на диск
- 🎨 **Удобный интерфейс** — крупные кнопки для сенсорных экранов

## 🚀 Быстрый старт (для новичков)

### Вариант 1: На своём компьютере (Windows/Mac/Linux)

1. **Скачайте проект** — нажмите зелёную кнопку "Code" → "Download ZIP"
2. **Распакуйте** в любую папку
3. **Установите PHP** (если нет):
   - Windows: скачайте с [php.net](https://windows.php.net/download/)
   - Mac: уже установлен
   - Linux: `sudo apt install php`
4. **Запустите** двойным кликом на `start.bat` (Windows) или в терминале:
   ```bash
   php -S localhost:8000
   ```
5. **Откройте браузер**: http://localhost:8000
6. **Пароль по умолчанию**: `password` (смените его!)

### Вариант 2: На хостинге (для доступа из интернета)

1. Купите любой дешёвый хостинг с PHP (от $1/месяц)
2. Загрузите все файлы через FTP
3. Откройте `generate_hash.php` и создайте свой пароль
4. Готово! Теперь доступно по адресу вашего хостинга

## 🔐 Первая настройка

1. Откройте `generate_hash.php` в браузере
2. Введите свой пароль (например: `moy_parol_123`)
3. Скопируйте полученный хеш
4. Откройте файл `config.php` блокнотом
5. Вставьте хеш вместо старого
6. **Удалите файл `generate_hash.php`** (важно для безопасности!)

# 🛠️ Техническая документация (для разработчиков)

## Архитектура

**SMFM** — ультралегкий файловый менеджер на чистом PHP и Vanilla JS без внешних зависимостей.

### Технический стек

- **Backend:** PHP 7.4+ (без фреймворков)
- **Frontend:** Vanilla JavaScript ES6, чистый CSS3
- **База данных:** Не требуется
- **Зависимости:** Отсутствуют
- **Размер:** ~17KB (CSS + JS)

### Ключевые возможности

#### 🔒 Безопасность
- **Bcrypt** хеширование паролей
- **CSRF** защита на всех формах
- **Brute-force protection** (задержка 2 сек)
- **Whitelist** валидация типов файлов
- **Path Traversal** защита
- **Санитизация** имён файлов
- **`.htaccess`** блокировка критичных файлов

#### 📁 Работа с папками
- Создание вложенных директорий
- Breadcrumbs навигация
- Рекурсивный расчёт размера
- Визуальное отображение структуры
- Умное переименование при конфликтах

#### 🚀 Chunked Upload
- Разбиение на части по 1 МБ
- Обход лимитов `post_max_size`
- Поддержка докачки (Resume)
- `flock` для целостности данных
- Drag & Drop для файлов и папок

#### 📱 UI/UX
- Адаптивный дизайн (mobile-first)
- Крупные элементы (44px+) для touch
- TV Mode с автоскрытием курсора
- Keyboard navigation
- Высокая контрастность

## Установка и настройка

### Требования
- PHP 7.4+
- Apache/Nginx с mod_rewrite
- 10 МБ свободного места

### Быстрая установка

```bash
# Клонировать репозиторий
git clone https://github.com/yelagin32/simple-magnitola-file-manager.git
cd simple-magnitola-file-manager

# Создать директории
mkdir uploads chunks
chmod 755 uploads chunks

# Сгенерировать хеш пароля
php generate_hash.php your_password

# Скопировать хеш в config.php
# Удалить generate_hash.php
rm generate_hash.php

# Запустить (для разработки)
php -S localhost:8000
```

### Конфигурация

Откройте `config.php`:

```php
// Хеш пароля (используйте generate_hash.php)
define('PASSWORD_HASH', '$2y$10$...');

// Директории
define('UPLOADS_DIR', 'uploads');
define('CHUNKS_DIR', 'chunks');

// Квота (0 = безлимит)
define('DISK_QUOTA_GB', 5);

// Размер чанка
define('CHUNK_SIZE', 1 * 1024 * 1024); // 1 MB
```

## API Endpoints

### POST /upload.php
Загрузка файла (chunked)

**Parameters:**
- `file` — файл (chunk)
- `chunk` — номер чанка (0-based)
- `chunks` — общее количество чанков
- `current_path` — путь для загрузки

**Response:**
```json
{
  "status": "success|error",
  "message": "..."
}
```

### POST /create_folder.php
Создание папки

**Parameters:**
- `folder_name` — имя папки
- `current_path` — текущий путь
- `csrf_token` — CSRF токен

### GET /delete.php
Удаление файла/папки

**Parameters:**
- `delete` — имя файла/папки
- `path` — текущий путь
- `csrf_token` — CSRF токен

## Безопасность

### Рекомендации для продакшена

1. **Смените пароль** через `generate_hash.php`
2. **Удалите** `generate_hash.php` после настройки
3. **Настройте HTTPS** (Let's Encrypt)
4. **Ограничьте доступ** по IP (в `.htaccess`)
5. **Регулярно обновляйте** PHP
6. **Мониторьте логи** (`error.log`, `upload.log`)

### .htaccess защита

```apache
# Блокировка конфигурации
<FilesMatch "^(config\.php|generate_hash\.php)$">
    Deny from all
</FilesMatch>

# Блокировка выполнения PHP в uploads
<Directory "uploads">
    php_flag engine off
</Directory>
```

## Производительность

### Оптимизация

- **Chunked upload** — обход лимитов сервера
- **flock** — предотвращение race conditions
- **Минимальный footprint** — 17KB (CSS + JS)
- **Без CDN** — нет внешних запросов
- **Кеширование** — статические ресурсы (1 месяц)

### Лимиты

Настройте в `.htaccess`:

```apache
php_value upload_max_filesize 2048M
php_value post_max_size 2048M
php_value max_execution_time 3600
php_value max_input_time 3600
```
